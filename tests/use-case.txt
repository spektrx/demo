Use-cases для информационной системы "Нарушениям.Нет":

1. **Регистрация пользователя:**
   - Акторы: Новый пользователь, Система.
   - Описание:
     1. Пользователь открывает страницу регистрации.
     2. Пользователь вводит уникальный логин, пароль (минимум 6 символов), ФИО, телефон и адрес электронной почты.
     3. Система проверяет валидность данных.
     4. Если данные введены корректно, пользователь заносится в базу данных.
     5. Система отображает подтверждение успешной регистрации.

2. **Авторизация пользователя:**
   - Акторы: Зарегистрированный пользователь, Система.
   - Описание:
     1. Пользователь открывает страницу авторизации.
     2. Пользователь вводит свой логин и пароль.
     3. Система проверяет введенные данные.
     4. Если данные верны, пользователь получает доступ к своему аккаунту.
     5. В случае неверного ввода данных, система предоставляет сообщение об ошибке.

3. **Просмотр заявлений пользователя:**
   - Акторы: Авторизованный пользователь, Система.
   - Описание:
     1. Пользователь авторизуется в системе.
     2. Пользователь переходит на страницу заявлений.
     3. Система отображает список заявлений пользователя с их статусами.
     4. Пользователь может просмотреть подробности каждого заявления.

4. **Создание нового заявления:**
   - Акторы: Авторизованный пользователь, Система.
   - Описание:
     1. Пользователь авторизуется в системе.
     2. Пользователь переходит на страницу формирования заявления.
     3. Пользователь указывает государственный регистрационный номер автомобиля и описание нарушения.
     4. Система проверяет заполненные данные.
     5. Если данные корректны, заявление добавляется в базу данных с статусом "новое".

5. **Управление заявлениями администратором:**
   - Акторы: Администратор, Система.
   - Описание:
     1. Администратор авторизуется в системе с логином copp и паролем password.
     2. Администратор переходит в панель администратора.
     3. Система отображает список всех заявлений.
     4. Администратор может изменить статус заявления на "подтверждено" или "отклонено".
     5. Система сохраняет изменения и уведомляет пользователя о новом статусе заявления.
